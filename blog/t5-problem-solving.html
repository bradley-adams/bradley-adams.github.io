Tell your non-tech friend a story about a time you:

1. Got blocked on a simple problem
2. Solved a problem in an elegant way
What was the problem?

In the colour exercise where you used Javascript to click on the circles and change their colours I got stuck on a couple of small things and I worked them out in a few different ways. The first one was for some reason when I wrote code it wouldn’t work versus copying code and then editing to reflect new colours it would work.

What problem solving techniques did you use?

Well I got involved in a bit of trial and error. I would put this under just giving something a go. I rewrote all code and it still didn’t work. I then started copying and pasting code that was already written to pin point the error. I found that when I deleted the calling of the code (or code that got the website started) it would work. From here learning that you should write this bit last.

How did you feel throughout the process?

I got quite nervous because what was being asked wasn’t complicated in terms of what I needed to do. I probably read to far ahead in instructions and then tried to do to many solutions at the same time. I was a good reminder that a lot of code is done in a specific order for a reason. I did feel great once I got it finished. Maybe even more because I managed to solve my own problem by myself and I also understood why.

What did you learn?

I learnt a lesson in testing as I go. Checking little bits are working correctly before moving onto the next problem. Otherwise you can write some much code and the mistake could be anywhere. I also learnt to stick to the designed process. Which seems straight forward given that most of what we are learning has prerequisites but I must have been rushing or looking to far ahead.
